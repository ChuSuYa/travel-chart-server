# TravelChart 服务端开发步骤

## 执行约定

- 所有 Git 提交信息, 代码注释和日志均使用中文. 标点统一使用英文.
- 每个步骤完成后必须进行编译, 容器启动, 健康检查和接口验证.
- 数据库变更必须同时提供新库初始化和存量库迁移方案.

## 步骤

| 步骤 | 目标 | 验收标准 | 状态 |
| --- | --- | --- | --- |
| 1 | 建立可复现的本地联调环境 | IDEA JDK 启动的 Spring 服务 8080-8086 可访问 | 已完成 |
| 2 | 验证管理服务最小闭环 | 管理员登录, 仪表盘, 用户状态和 POI 状态接口可用 | 已完成 |
| 3 | 统一移动端 P0 API 契约 | 用户, 行程, 天气, 社交接口通过契约测试 | 未开始 |
| 4 | 建立自动化质量门禁 | Maven 测试, 前端构建和接口冒烟测试可在 CI 执行 | 未开始 |

## 当前阻塞

- 本地联调使用 IDEA 自带 JDK 和 Maven, Spring 服务 8080-8086 已成功启动.
- Docker Compose 仍需处理已有基础容器缺少健康检查元数据的问题, 再纳入 CI 环境.
- 首次启动管理服务前必须设置 `ADMIN_BOOTSTRAP_USERNAME` 和 `ADMIN_BOOTSTRAP_PASSWORD`.
- 已有数据库需要执行 `db/migrations/V2__manage_module.sql`.
